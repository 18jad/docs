From version **TBD**, you can use `client` [Prisma Client extensions](/concepts/components/prisma-client/client-extensions) to add top-level methods to Prisma Client.

To create Prisma Client extensions, use the [`$extends`](/concepts/components/prisma-client/client-extensions#about-prisma-client-extensions) client-level method. Use the `client` extension type to extend Prisma Client.

</TopBlock>

## Enable the preview feature

Before you create Prisma Client extensions, you must enable the `clientExtensions` feature flag in the `generator` block of your `schema.prisma` file, as follows:

```prisma
generator client {
  provider        = "prisma-client-js"
  previewFeatures = ["clientExtensions"]
}
```

## Extend Prisma Client

To add methods to Prisma Client, use the following structure:

```ts
const xprisma = prisma.$extends({
  client?: { ... }
})
```

- `client`: An optional field that adds a top-level method to the client

The following example uses a `client` extension to start an [interactive transaction](/concepts/components/prisma-client/transactions#interactive-transactions-in-preview) without callbacks.

```ts
const xprisma = prisma.$extends({
  client: {
    method(args:) { ... }, // the interactive transaction code goes inside the curly braces
  }
})

xprisma.$method
```

When you call a method in an extension, use the constant name from your `$extends` statement, not `prisma`. In the above example, `xprisma.$method` works, but `prisma.$method` does not.

The following code then starts an interactive transaction without the traditional callback:

```ts

const tx = await xprisma.$begin()

await tx.user.create(...)
await tx.user.update(...)

await tx.$commit()

```
